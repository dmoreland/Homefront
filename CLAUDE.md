# Home Front — WW2 industrial idle game

A grand-strategy-inspired WW2 idle/clicker. A React (Vite) app with a faction-grouped nation picker — seven nations: Allies (UK, USA, France), Axis (Germany, Italy, Japan), and the Soviet Union. Prototyped in Claude.ai as a single file, since split into pure game logic, a `useGameEngine` hook, and presentational components.

## How to run
```
npm install
npm run dev     # local dev server
npm test        # vitest unit + render tests
npm run build   # production build
```
Deploy: GitHub Pages via `.github/workflows/deploy.yml` (static Vite build, no backend); live at https://dmoreland.github.io/Homefront/.

## Project layout
- `src/data/gameData.js` — shared tables (RES, LINES, FRESH). Universal across nations.
- `src/data/nations.js` — the `NATIONS` config array (7 nations) + `getNation`/`newGame` + `mkUpgrades` (War Cabinet builder: radar + laws) + `mkFocuses` (National Focus builder: shared industry spine + per-nation identity focuses). Each nation is a data object: `faction` (Allies/Axis/Soviet, for picker grouping), starting resources, passive trickle, generators (incl. unique buildings like the Synthetic Refinery), forces, theatres, upgrades, `focuses`, and `theme` (Scene palette). **Adding a nation is a config change here** — validate it with `npm run balance`.
- `src/game/` — pure, unit-tested logic: `simulate.js` (the nation-parameterised game step), `economy.js` (cost scaling/affordability), `missions.js` (theatre resolution), `offline.js` (offline earnings), `theatres.js` (`theatreDuration`), `forces.js` (readiness / equipment-quality), `focus.js` (National Focus: timed per-nation tech tree), `pressure.js` (enemy pressure + garrison), `saveStore.js` (persistence adapter). Co-located `*.test.js` files.
- `src/hooks/useGameEngine.js` — owns state and the impure lifecycle (tick, autosave, load/offline, actions, nation selection). The one place timers/storage/toasts live.
- `src/components/` — presentational pieces: `NationPicker` (start screen), `GameView` (active campaign composition), and the panels (Scene, ResourceBar, GeneratorList, ProductionLines, ForcesList, Theatres, FocusTree, WarCabinet, Foundry, Header, Footer, Toast).
- `src/ui/` — shared `format.js` (fmt, costStr) and `styles.js`.
- `src/App.jsx` — picks NationPicker vs GameView, mounts the toast.

## Nation config model
`simulate(state, dt, nation)` and the actions read everything from the active nation object, so nations are pure data. Key shapes (see `nations.js` header comment): generators use `produces` (scaled by civ & theatre multipliers), `globalMult` (Civilian Factory), or `converts` (Synthetic Refinery: throttles like a production line, trading steel for oil/rubber). Theatre rewards are `{ kind: "mult", res, per }` or `{ kind: "flat", per }` applied per victory. Laws carry `manpowerMult`, speed upgrades `speeds`+`factor`, the foundry upgrade `tapMult`. Force slots (inf/arm/air/fleet) and equipment keys are stable across nations; only names/costs differ. Germany's identity: strong steel, no colonial trickle, oil/rubber only via Synthetic Refineries or seized in theatres; forces are the three service branches (Wehrmacht/Panzer, Luftwaffe, Kriegsmarine) — no SS/political units.

## Core design
Two-stage production chain (the grand-strategy hook — resources do NOT buy units directly):
1. **Raw resources** (steel, aluminium, oil, rubber, manpower) from generators; UK gets a passive Empire trickle of +0.2 oil & rubber/sec.
2. **Production lines** CONSUME resources per second to make equipment (rifles, artillery, tanks, fighters, warships). Lines throttle proportionally when starved and show STALLED %.
3. **Forces** recruited from equipment + manpower: infantry/armoured divisions, air wings, fleets. Air & fleets have oil upkeep.
4. **Theatres** (e.g. Battle of Britain / Atlantic / North Africa): commit forces to timed operations. Victory returns forces, grants stacking oil/rubber/aluminium bonuses + War Score. Stage requirements scale ×2 per victory, duration ×1.3. **No theatre boosts steel** — steel scaling is industry-driven (below), so no single theatre is a must-rush. **Enemy pressure** (the opponent): once a front is engaged (≥1 victory) it accrues pressure over time; when it maxes an enemy offensive lands, and its outcome (repel / contain / overrun) is set by the front's **garrison** (idle forces you station there) × readiness vs the assault. Winning an operation resets a front's pressure; garrisoning idle forces holds a front you've moved on from. Config/logic in `src/game/pressure.js`.
5. **National Focus** tree (per-nation, per-run): a HOI4-style tree of focuses that complete on a **timer** (one active at a time) and grant permanent-for-the-run bonuses. A shared industry spine (foundry tap ×3 → steel +25% → +40% → all +25% — the folded former War-Cabinet industry tiers, now **time-gated** and the main in-run steel-scaling lever) plus per-nation identity focuses (e.g. USSR "Not One Step Back" manpower ×2, Japan "Strike South" flat oil, USA "Arsenal of Democracy" +30% all). Config in `src/data/nations.js` (`focuses`), logic in `src/game/focus.js`.
6. **War Cabinet** upgrades: War-Score-bought conscription laws (manpower ×2/×4/×8) and Radar (air/naval ops 25% faster). (Industry/steel scaling moved to the National Focus tree.) The Civilian Factory (generator) is the repeatable +10%-all multiplier.
7. **Prestige & doctrines** (cross-run, shared across nations): reach Total Victory (every theatre at Stage 3+) to prestige — bank Doctrine Points (`floor(1.5·√warTotal)`), wipe the campaign, return to the picker. Points buy permanent nodes in the Doctrine HQ across four branches — **War Economy** (resource output, starting stockpile, offline rate), Land, Sea, Air. Config in `src/data/doctrines.js`; effects aggregate into a `mods` bundle (`src/game/doctrines.js`).

## Architecture notes
- `simulate(state, dt, nation, mods?)` (in `src/game/simulate.js`) is the PURE game step — used by the 250ms tick AND offline earnings (50% rate + doctrine bonus, 8h cap). Keep it pure; nation balance lives in `src/data/nations.js`, doctrine balance in `src/data/doctrines.js`. `mods` (from `computeMods`) is optional — omitting it means no doctrine effects. Mission and offline resolution are likewise pure (`game/missions.js`, `game/offline.js`) and unit-tested.
- Doctrines: `game/doctrines.js` has `computeMods` (purchased → `mods`), `doctrinePoints`, `totalVictory`, `effectiveForceCost`, `effectiveNeed`. `mods` carries genMult/forceCostMult/upkeepMult/stageReqDelta/opSpeedMult/offlineRateAdd/startBonus/tapMult **plus** manpowerMult/theatreRewardMult/flatGen (dims also produced by National Focus), and is threaded into simulate, `theatreDuration`, and the recruit/launch/tap actions. `game.warTotal` is the cumulative War Score that drives the prestige payout (separate from spendable `warScore`).
- National Focus: `game/focus.js` is pure — `computeFocusMods(nation, done)` aggregates a nation's completed focuses into a mods-shaped bundle; `mergeMods(a, b)` folds it on top of the doctrine bundle (multiply multiplicative dims, add/sum additive ones); `resolveFocus(game, nation, nowMs)` completes the active focus when its absolute `endsAt` elapses (offline-safe, mirrors `resolveMissions`), applying any one-time `grant`; `focusAvailable` gates the start. State is `game.focus = { active: {id, endsAt} | null, done: {} }` — **per-run** (reset on prestige/reset). The engine computes `mods = mergeMods(computeMods(doctrines), computeFocusMods(nation, game.focus.done))`, so focus completion re-derives mods. Effects reuse the doctrine vocabulary plus `manpowerMult`/`flatGen`/`theatreReward`/`grant`. Steel scaling now flows through `mods.genMult` (the industry-spine focuses) — simulate no longer reads per-upgrade `outputMult`. The balance sim (`scripts/balance.mjs`) drives focuses greedily (industry spine first) and recomputes mods each second.
- Enemy pressure: `game/pressure.js` is pure — `stepPressure(game, nation, dt)` advances `game.pressure[thId]` (0..1) on every **engaged** front (stages≥1), resolving **at most one offensive per front per call** (offline-safe: a huge `dt` can't cascade). `defenseStrength(garrison, readiness, nation)` = Σ garrison count × pool readiness × grit; `assaultStrength(stage)` escalates with victories; `offensiveOutcome(D, A)` is a **deterministic** tier from the D/A ratio (no RNG) → repelled (garrison ~10% losses) / contained (front held, ~30% losses, no rollback/raid) / overrun (undefended: stage −1, resources raided, ~60% losses). Garrison slows pressure growth. State is `game.pressure = { [thId]: 0..1 }` + `game.garrison = { [thId]: { inf, arm, air, fleet } }` — **per-run**; garrisoned forces are drawn out of the free `game.forces` pool (the offense↔defense tension). Threaded into the engine tick and offline load beside `resolveMissions`/`resolveFocus`; a mission **victory zeroes that front's pressure** (`resolveMissions`). The balance sim (`scripts/balance.mjs`) garrisons every engaged front it isn't actively assaulting (to repel, using the front's own troop types) so pressure never rolls back a won stage.
- Resource bar shows NET flow (gen − line consumption − upkeep); deficit cards go red with ▼. Consumption is measured at current throttle, so a fully stalled line under-reports its theoretical demand.
- Missions store absolute `endsAt` timestamps → they complete correctly across offline periods; resolved on load and in tick. **F15 fuel mechanic**: when oil runs dry (`isFuelStarved`: stock 0 and net < 0), each live tick pushes active air/naval missions' `endsAt` forward by 50% of the tick (`applyFuelPenalty` in `game/theatres.js`), so they run at half speed until fuel recovers. Land ops are unaffected, and the penalty only applies to live ticks (offline resolution stays absolute-timestamp simple). The UI shows a ⛽ warning on slowed ops.
- **Readiness (equipment quality)** — the quality-vs-quantity layer (`game/forces.js`, pure). A force can be **recruited** fully-equipped (readiness 1.0) or **mobilised** with whatever equipment is on hand at **any** readiness (even 0%) — mobilise charges full manpower but only an `r`-fraction of each equipment (`r` = the tightest equipment ratio); the only gate is manpower. `game.readiness[forceId]` holds the count-weighted pool average, updated by `poolReadiness` on each recruit/mobilise. **Reinforcement** (`reinforce`, run inside `simulate` each tick and offline): surplus equipment is poured into under-strength pools — lifting readiness toward full — *before* it's free to raise new units. Pools hold equipment uniformly across their types (invariant `held_k = count·readiness·cost_k`, preserved by recruit/mobilise/reinforce), so a lift of Δ costs `count·Δ·cost_k` of each key; pools fill in force-config order, so shared equipment (e.g. artillery) reinforces earlier forces first. At launch, `committedReadiness(need, readiness)` sets the operation's readiness, which (a) multiplies duration via `attritionMult` (slower when under-equipped) and (b) rolls `willFail = Math.random() < failureChance(...)` **once**, baked onto the mission so resolution stays pure; the per-force readiness is stored as `m.forceReadiness`. `resolveMissions` reads `m.willFail`: victory → stage+score+forces returned; **defeat** → no stage/score, only `(1 − FAIL_LOSS)` of committed forces survive. Returning units **blend** back into their pool via `mergeReadiness(before, poolR, ret, m.forceReadiness[k])`, so reinforcement done while they were deployed isn't lost or handed out free. One per-nation trait differentiates play: `grit` (higher = gentler attrition + lower defeat threshold — quantity over quality; default 0.5, set per nation in `nations.js`). The balance simulator recruits fully-equipped, so it never triggers reinforcement/attrition/defeat.
- Persistence: `saveStore` adapter tries `window.storage` (Claude artifact API) then falls back to `localStorage` (deployed web). Two independent stores: the campaign save (`home-front-save`, carries `nationId`, cleared on reset/prestige) and the cross-run doctrine save (`home-front-doctrines`, survives resets). Autosave 10s + visibilitychange.
- Scene (`components/Scene.jsx`) is inline SVG built from the game-icons.net silhouettes (imported `?raw`, nested as `<svg viewBox="0 0 512 512">` and tinted via `currentColor` from the nation `theme`): factory/refinery emblems rise with industry, tanks & infantry muster on the land with your divisions, fighters patrol the sky with air wings, the fleet stands out to sea; SMIL animation (animateMotion/animate/animateTransform). Purely a reflection of game state.

## Balance philosophy
First prestige should take 30–60 min; each subsequent run ~2× faster to the same point. Watch the mid-game around the tank/fighter line costs — that's where the interesting scarcity decisions live. All pacing knobs are constants in the data tables.
